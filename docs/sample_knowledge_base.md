# AI Engineer Hiring Notes

AI Engineer candidates should be comfortable building applied LLM systems, not only writing prompts.

Important skills include Python, FastAPI, embeddings, vector search, retrieval augmented generation, evaluation pipelines, and Docker.

Strong candidates can explain how they reduce hallucinations. Common techniques include citing retrieved sources, checking answer groundedness, logging traces, measuring retrieval quality, and testing prompts on representative examples.

Production AI systems often need observability. Useful metrics include latency, cost, retrieval relevance, answer groundedness, and user feedback.

Agentic systems are useful when the workflow has distinct responsibilities such as planning, retrieval, answer generation, critique, and report generation.
